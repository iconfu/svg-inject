SVGInject v2 Examples

Everything below is impossible with a regular <img> tag. These examples only work because SVGInject replaces the <img> with an inline <svg>, giving CSS and JavaScript full access to the SVG internals.

Note: This page must be served via HTTP, not opened from the file system. Run npx serve . from the project root and open http://localhost:3000/examples/.

Change fill color with CSS

A normal <img> ignores the CSS color property. An inline SVG using fill="currentColor" inherits it. Same SVG file, different CSS class.

color: coral
color: teal
color: indigo
color: goldenrod
color: steelblue
color: tomato
color: orchid

With <img>, you'd need a separate image file for each color.

Change color on hover

CSS :hover can change the fill color of an inline SVG. Impossible with <img> — you can scale or rotate an image on hover, but you can't change its colors.

gray → crimson

CSS custom properties (variables)

SVGs can reference CSS variables like fill: var(--icon-fill). Different CSS classes set different variable values. Completely impossible with <img>.

ID conflict prevention

This SVG uses a <linearGradient id="grad1">. Injected three times without SVGInject's ID uniquification, only the first would show the gradient — the other two would reference the wrong ID. SVGInject appends unique suffixes so all three render correctly.

--inject-1
--inject-2
--inject-3

Automatic accessibility

SVGInject reads the alt attribute from the <img> and sets the correct ARIA roles on the injected SVG. Inspect the DOM to see the attributes.

alt="Team of business people"
→ role="img" aria-label
alt="Angel icon"
→ role="img" aria-label
alt="" (decorative)
→ role="none" aria-hidden
alt="" (decorative)
→ role="none" aria-hidden
